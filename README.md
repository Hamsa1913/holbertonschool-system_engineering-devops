# Web Infrastructure Design

This project is part of the Holberton School System Engineering & DevOps curriculum.

The objective of this project is to design different web infrastructure architectures and understand how each component contributes to building a reliable, scalable, secure, and highly available web application.

The diagrams were created using **draw.io** and exported as PNG images before being uploaded to an image hosting service, as required by the project.

---

## Learning Objectives

By completing this project, I learned how to:

- Design a simple web infrastructure
- Design a distributed web infrastructure
- Build a secured and monitored infrastructure
- Scale a web infrastructure
- Understand the responsibilities of each infrastructure component
- Explain system redundancy and high availability
- Identify Single Points of Failure (SPOF)
- Understand load balancing strategies
- Explain database replication
- Understand HTTPS, firewalls, and monitoring systems

## Tasks

| Task | Description |
|------|-------------|
| **0** | Design a simple web stack using one server with a LAMP architecture. |
| **1** | Design a distributed web infrastructure using multiple servers and a load balancer. |
| **2** | Improve the infrastructure by adding HTTPS, firewalls, and monitoring. |
| **3** | Scale the infrastructure by separating components and clustering the load balancers. |
